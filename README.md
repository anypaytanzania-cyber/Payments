# 🚀 AnyPay Tanzania API Documentation

Welcome to the official API documentation for **AnyPay Tanzania**.
This API enables developers to integrate **mobile money payments, order creation, and payment tracking** seamlessly.

---

## 🔐 Authentication

All API requests must include the following headers:

```http
Authorization: Bearer <your_access_token>
API-Key: <your_api_key>
Content-Type: application/json
```

### Example (Python)

```python
headers = {
    "Authorization": "Bearer YOUR_ACCESS_TOKEN",
    "API-Key": "YOUR_API_KEY",
    "Content-Type": "application/json"
}
```

---

## 📌 Base URL

```
https://anypaytanzania.com/api/payments/
```

# 💰 1. Wallet Pull Payment

Initiate a **mobile money payment request** to a customer.

### Endpoint

```
POST /wallet/pull/
```

### Request

```json
{
  "order_id": "ORD-8442355",
  "phone": "0798100380",
  "amount": 1000
}
```

### Response

```json
{
  "status": "success",
  "resultcode": "000",
  "message": "Request in progress. You will receive a callback shortly",
  "order_id": "ORD-8442355",
  "transid": "ORD-8442355",
  "msisdn": "255798100380",
  "payment_reference": "WQ20260428BBD25BEA4407322A",
  "order_response": {
    "result": "SUCCESS",
    "reference": "S20523452275"
  },
  "wallet_response": {
    "result": "SUCCESS",
    "reference": "S20523452284",
    "message": "Wallet push successful"
  }
}
```

### Notes

* Accepts local numbers (`07XXXXXXXX`)
* Automatically converts to international format (`255XXXXXXXXX`)
* Payment is **asynchronous** (user confirms on phone)

# 🧾 2. Create Order (Minimal)

Generate a payment link for customer checkout.

### Endpoint

```
POST /create-order-minimal/
```

### Request

```json
{
  "buyer_phone": "0798100380",
  "amount": 1000
}
```

### Response

```json
{
  "order_id": "ORD-ac00cd6466",
  "reference": "S20523453011",
  "result": "SUCCESS",
  "payment_token": "63831626",
  "payment_url": "https://tz.selcom.online/paymentgw/checkout/...",
  "redirect": "http://anypaytanzania.com/payment/success"
}
```

### Usage

* Redirect user to `payment_url`
* Supports mobile money & cards

# ⚡ Payment Flow

## Wallet Pull Flow

1. Create order (optional)
2. Call `/wallet/pull/`
3. Customer receives prompt
4. Customer enters PIN
5. Confirm via callback or status check

## Hosted Payment Flow

1. Call `/create-order-minimal/`
2. Redirect user to payment page
3. Customer completes payment
4. Redirect to success page

# 💸 5. Wallet Payout

Initiate a **mobile money payout** to a customer's mobile wallet.

This endpoint allows merchants to transfer funds directly to a recipient's mobile money account across supported networks.

### Endpoint

```http
POST /wallet/payout/
```

### Request

```json
{
  "order_id": "ORD-188890",
  "recipientNames": "Joseph",
  "amount": 1000000,
  "serviceCode": "TZ-TIGO-B2C",
  "phone": "0713570435",
  "callbackUrl": "https://anypaytanzania.com/api/payments/webhook-tembo-payout"
}
```

### Request Parameters

| Field          | Type   | Description                                       |
| -------------- | ------ | ------------------------------------------------- |
| order_id       | String | Unique payout reference generated by the merchant |
| recipientNames | String | Recipient's full name                             |
| amount         | Number | Amount to be sent                                 |
| serviceCode    | String | Mobile money provider code                        |
| phone          | String | Recipient phone number                            |
| callbackUrl    | String | URL that receives payout status notifications     |

### Successful Response

```json
{
  "success": true,
  "message": "Payout initiated successfully",
  "order_id": "ORD-18890",
  "payment_reference": "WQ20260617587366CA3564670A",
  "status": "processing",
  "amount": "1000",
  "currency": "TZS",
  "fee": "50.00",
  "fee_percent": "5.00",
  "fixed_fee": "0",
  "total_debited": "1050.00",
  "recipient_name": "Joseph",
  "recipient_phone": "255713570435",
  "service_code": "TZ-TIGO-B2C",
  "provider_transaction_id": "I8fOXEtT5qg8",
  "provider_status": "PAYMENT_ACCEPTED",
  "provider_response": {
    "statusCode": "PAYMENT_ACCEPTED",
    "transactionRef": "ORD-18890",
    "transactionId": "I8fOXEtT5qg8"
  }
}
```

### Duplicate Request Response

Returned when the same `order_id` has already been processed.

```json
{
  "success": true,
  "message": "Duplicate request",
  "payment_reference": "WQ20260615FDB7F5D230A39797",
  "status": "failed"
}
```

### Status Definitions

| Status     | Description                                   |
| ---------- | --------------------------------------------- |
| processing | Payout request accepted and being processed   |
| completed  | Funds successfully delivered to recipient     |
| failed     | Payout failed or rejected                     |
| duplicate  | Request with the same order ID already exists |

### Notes

* Accepts local phone numbers (`07XXXXXXXX`)
* Automatically converts phone numbers to international format (`255XXXXXXXXX`)
* Payout processing is **asynchronous**
* Funds are sent directly to the recipient's mobile wallet
* Transaction charges are calculated automatically
* `fee` represents the transaction charge applied
* `total_debited` represents the amount sent plus transaction fees
* `payment_reference` should be stored for reconciliation and tracking
* `provider_transaction_id` is the transaction identifier generated by the mobile money provider
* Final payout status will be delivered to the supplied `callbackUrl`

### Example Service Codes

| Service Code  | Provider     |
| ------------- | ------------ |
| TZ-TIGO-B2C   | Tigo Pesa    |
| TZ-VODA-B2C   | M-Pesa       |
| TZ-AIRTEL-B2C | Airtel Money |
| TZ-HALO-B2C   | HaloPesa     |

### Payout Flow

1. Submit payout request.
2. System validates recipient details.
3. Phone number is converted to international format.
4. Available balance is verified.
5. Transaction fee is calculated.
6. Payout request is submitted to the mobile money provider.
7. Provider returns a transaction reference.
8. Payout enters processing state.
9. Final status is delivered through the callback URL.
10. Recipient receives funds in their mobile wallet.

# 🛠 Best Practices

* Always verify payments using `/check-order-status/`
* Do not rely only on frontend confirmation
* Store `order_id` and `reference`
* Handle asynchronous responses properly

# 🧪 Testing Tips

* Use small amounts (e.g., 500 TZS)
* Use valid Tanzanian numbers
